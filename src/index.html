① AI가 문제 푸는 순서 — 문제 해결 7단계

  1. 데이터 수집 및 불러오기
  2. 데이터 전처리 및 탐색적 데이터 분석
  3. 데이터를 훈련 데이터와 테스트 데이터로 분할 (train_test_split)
  4. 데이터 스케일링을 통해 Train 데이터의 값의 범위를 조정 (Scaler 활용)
  5. 다양한 머신러닝 알고리즘 및 인공신경망을 적용해 훈련 데이터로 모델 학습

    🟠 지도학습 — 예측 (회귀)

    숫자(연속값)를 맞히는 회귀. 직선(선형)부터 곡선(다항), 과대적합을 막는 규제 모델까지 같은 회귀 가족이에요.

    🟠 지도학습 — 분류

    🟠 체험존 ① — kNN: k 값에 따라 답이 바뀐다!

    가운데 점(?)은 아직 종류를 모르는 새 데이터예요. 가장 가까운 이웃 k개에게 물어봅시다. k 값을 바꿔 보세요!

    🟠 체험존 ② — 의사 결정 트리: 스무고개로 동물 찾기

    간단한 예시 트리예요. 질문에 답하며 가지를 따라 내려가 보세요. (매·펭귄·돌고래·곰 중 하나!)

    🟠 체험존 ③ — 나이브 베이즈: 스팸 메일 필터 🎯

    나이브 베이즈(위 분류 카드)는 조건부 확률로 판단해요. 아래에서 스팸 필터가 확률을 어떻게 계산하는지 직접 만져 보세요.

    기호이름
    P(A|B)조건부 확률
    P(B)사전 확률(Prior)
    P(B|A)사후 확률(Posterior)
    P(A)증거(Evidence)

    🎮 직접 계산해 보기 — 스팸 메일 필터

    "무료"라는 단어가 들어 있는 메일이 도착했어요. 슬라이더로 확률을 바꿔 보며, 이 메일이 스팸일 사후 확률 P(스팸|무료)가 어떻게 변하는지 관찰하세요.

    ※ 증거(Evidence) P(무료) = P(무료|스팸)·P(스팸) + P(무료|정상)·P(정상) 으로 계산됩니다.

    🟢 앙상블 학습

    여러 모델을 합쳐 더 강한 모델을 만드는 기법. 합치는 방식에 따라 배깅·부스팅·스태킹 세 갈래로 나뉩니다 — 카드 아래 구분표로 정리!

    계열합치는 방식대표 알고리즘실생활 비유
    배깅 (Bagging)
    병렬 · 동시에    		 같은 종류의 모델 여러 개를 서로 다른 데이터 조각으로 동시에 학습시키고, 결과를 다수결·평균으로 합침 랜덤 포레스트 반 전체가 각자 풀고 투표 🗳️
    부스팅 (Boosting)
    직렬 · 차례로    		 앞 모델이 틀린 문제를 다음 모델이 이어받아 집중 보완하며 차례로 학습 Adaboost, XGBoost 오답 노트로 복습하는 전교 1등 📓
    스태킹 (Stacking)
    2단계 · 종합    		 서로 다른 모델들의 예측값을 모아, 2단계 최종 모델이 그 예측 패턴을 다시 학습해 결론 Stacking 우등생 의견을 종합하는 담임 🎓

    💡 구분 문제 힌트: "동시에 여러 개" = 배깅, "실수를 이어받아 차례로" = 부스팅, "예측값을 다시 학습" = 스태킹!

    🔵 비지도 학습

    정답(레이블) 없이 데이터 스스로 구조를 찾는 학습. 군집화(K-Means·DBSCAN)와 차원 축소(PCA)로 나뉘어요.

    🍩 같은 '도넛 모양' 데이터, 누가 잘 나눌까? — K-Means vs DBSCAN
    ❌ K-Means (거리 기반)
    ✅ DBSCAN (밀도 기반)
    👨‍🏫 선생님의 한 줄 요약
    데이터가 동글동글 예쁘게 뭉쳐있을 땐 K-Means, 뱀처럼 길게 휘어있거나 노이즈가 많을 땐 DBSCAN, 변수가 너무 많아 머리가 아플 땐 PCA로 먼저 압축하기!

    🕹️ 강화학습 — 정답 대신 '보상'으로 배운다

    지도학습(정답지 보고 공부)·비지도학습(정답 없이 패턴 찾기)에 이은 세 번째 학습 방법이에요. 정답 레이블 대신 행동의 결과로 받는 보상과 벌점이 선생님 역할을 합니다.

    학습 방법어떻게 배우나대표 예
    지도학습정답(레이블)이 달린 데이터로 공부 — 문제집과 답지분류·회귀 (kNN, 선형 회귀…)
    비지도학습정답 없이 데이터 속 구조·패턴을 스스로 발견군집화·차원 축소 (K-Means, PCA…)
    강화학습행동해 보고 돌아오는 보상/벌점으로 시행착오하며 최선의 전략 학습알파고, 게임 AI, 로봇 보행
    👨‍🏫 선생님의 한 줄 요약
    답지를 보며 공부하면 지도, 답 없이 스스로 무리를 찾으면 비지도, 해보고 간식과 벌점으로 배우면 강화!

    ③ 데이터 다듬기 — 스케일링

    키(150~190)와 신발 사이즈(230~290)처럼 변수마다 값의 범위가 크게 다르면, 범위가 큰 변수가 학습을 지배해 버려요. 이를 막고 계산도 쉽게 하도록 서로 다른 변수의 값 범위를 일정하게 맞춰 주는 방식을 스케일링이라 합니다. (변수 개수를 줄이는 PCA의 '차원 축소'와는 다른 개념이에요.) 대표적인 방법은 아래와 같아요.

    방법설명
    StandardScaler()데이터를 정규분포(평균 0, 표준편차 1) 형태로 변환
    MinMaxScaler()데이터를 기본값 [0,1] 사이 숫자로 변환 (옵션 지정 시 [-1,1] 등)
    RobustScaler()중앙값(Median), IQR(상위25%-하위25%)을 사용해 변환

    ④ 딥러닝 입문 — 퍼셉트론부터 신경망 종류까지
    
  
    
  
    1. 뇌를 흉내 낸 퍼셉트론 & 순전파·역전파
    
  
    
    
    퍼셉트론: 인간의 뉴런을 모방한 인공신경망을 구성하는 기본 단위. 편향은 가중합 Σ(wx)을 구한 다음 더해짐
    
    
    구조: 입력(x₁..xₙ) → 각 가중치(w₁..wₙ) → Σ(합산, +편향 b) → 계단함수(step) → out(t)
    
    
    
      
      
      
    
    
    
    버튼을 눌러 데이터의 흐름을 관찰하세요.
    
    
    
    
      
    
        순전파 — 입력한 데이터에 가중치(w)를 곱하고 편향(b)을 더한 뒤(y = wx + b), 활성화 함수를 거쳐 최종 결괏값을 계산하는 과정
      
    
      
    
        역전파 — 실제 정답과 모델의 예측값의 차이인 오차를 계산하고, 오차를 줄이기 위해 앞의 층으로 돌아가며 가중치와 편향을 수정하는 과정
      
    
    
    
    
    
      
    
      
    
    
    
  
    
  
    

  
    
  
    2. 신경망 낱말 사전 — 핵심 용어
    
  
    
    
    용어정의
    
    
    퍼셉트론인간의 뉴런을 모방한 인공신경망을 구성하는 기본 단위. 편향은 가중합 Σ(wx)을 구한 다음 더해짐
    
    
    인공신경망다층 퍼셉트론으로 1개 이상의 은닉층이 연결된 신경망
    
    
    딥러닝인공신경망을 여러 층으로 깊게(Deep) 쌓아 데이터를 학습하는 머신러닝의 분야
    
    
    은닉층인공 신경망의 여러 층 중에서 입력층과 출력층 사이에 있는 층
    
    
    가중치입력값이 결과에 미치는 중요도(영향력)를 결정하는 수치
    
    
    편향입력값과 관련없이 퍼셉트론이 얼마나 쉽게 활성화(출력 신호를 보낼지)될지 조절하는 값. 편향이 크면 쉽게 활성화, 작거나 음수면 활성화되기 어려움
    
    
    순전파입력한 데이터에 가중치(w)를 곱하고 편향(b)을 더한 뒤(y = wx + b), 활성화 함수를 거쳐 최종 결괏값을 계산하는 과정
    
    
    역전파실제 정답과 모델의 예측값의 차이인 오차를 계산하고, 오차를 줄이기 위해 앞의 층으로 돌아가며 가중치와 편향을 수정하는 과정
    
  
    
  
    
    
    인공신경망 구조: Input(3 노드) → Hidden(4 노드, 서로 완전연결) → Output(2 노드)
    
    
  
    
  
    

  
    
  
    3. 켤까 말까? 활성화 함수
    

  
    
    활성화 함수란? 인공신경망(Artificial Neural Network)에서 노드의 입력값을 변환하여 다음 층으로 전달할지, 그리고 어떤 크기로 전달할지 결정하는 수학적 함수
  
    

  
    
    
    
      
    🔚 출력층용
    
      
    
        
    함수사용 상황
    
        
    없음/선형(Linear)회귀(예측) 분석 시 출력층에 사용
    
        
    시그모이드(Sigmoid)둘 중 하나의 정답을 선택하는 이진 분류
    
        
    소프트맥스(Softmax)3개 이상의 선택지 중 하나의 정답을 선택하는 다중 클래스 분류
    
      
    
    
    
    
    
      
    🔒 은닉층용
    
      
    
        
    함수설명
    
        
    계단 함수가중합에 편향을 더한 값이 0보다 크면 1, 아니면 0 (초기 퍼셉트론 구조)
    ⚠️ 미분이 안 돼서 역전파(오차 수정)가 불가능 → 요즘 딥러닝 은닉층에선 쓰지 않는 역사 속 원형이에요. 그 자리를 물려받은 게 아래 ReLU 가족!
    
        
    ReLU은닉층에서 자주 사용되며 입력값이 양수면 그대로, 0 이하면 0으로 출력
    
        
    Tanh하이퍼볼릭 탄젠트. 출력값이 -1 ~ 1 사이이며 데이터의 중심을 0으로 맞춤
    
        
    Leaky ReLUReLU에서 0을 출력한 노드의 비활성화 문제를 막기 위해 씀. 0 미만의 음수 입력값에 아주 작은 수(α)를 곱해 아주 작은 음수를 출력함
    
        
    ELU입력값이 0 이상이면 그대로 출력, 0 미만이면 α(eˣ−1) 값 출력
    
      
    
    
    
  
    

  
    
    
    🎮 곡선을 직접 움직여 보세요 (체크박스로 함수 켜고 끄기 + 슬라이더로 x 이동)
    
    
    
    
    
    
    현재 입력 x = 1.00
    
    
    
    
    ※ 그래프의 Leaky ReLU는 눈에 잘 보이도록 α=0.1로 그렸어요. (코드 예제에서는 보통 α=0.01) 회귀 출력층에서 쓰는 선형(Linear) 함수도 체크박스로 켜 볼 수 있어요.
    
  
    

  
    
    
    🤔 Softmax는 왜 위 그래프에 없을까?
    
    
    Softmax는 입력 하나가 아니라 여러 개의 출력값(z)을 동시에 받아, 전체 합이 1이 되는 확률로 바꿔 주는 함수예요. 그래서 단일 곡선으로는 그릴 수 없습니다. 아래에서 클래스 3개의 출력값을 직접 조절해 보세요 — 세 확률의 합은 항상 100%!
    
    
    
    
    
    
    
    
    
    
    
    
    
    
    가장 큰 출력값을 가진 클래스가 가장 높은 확률을 갖게 되고, 모델은 그 클래스를 정답으로 선택합니다. 이것이 "3개 이상의 선택지 중 하나의 정답을 선택하는 다중 클래스 분류"에 Softmax를 쓰는 이유예요.

    📊 분류 평가지표 — 정확도만 믿으면 안 되는 이유
    
  
    
    
    분류 모델의 예측 결과는 아래 혼동 행렬(Confusion Matrix) 네 칸으로 나뉘어요. (예: 암 진단 AI)
    
    
    
      
    예측: 암 ⚠️예측: 정상 ✅
    
      
    실제: 암TP 진짜 양성
    잘 잡아냈다! 👍    		FN 거짓 음성
    환자를 놓쳤다 😱
    
      
    실제: 정상FP 거짓 양성
    멀쩡한 사람을 오진 💦    		TN 진짜 음성
    정상을 정상으로
    
    
    
    
    
      
    지표뜻계산
    
      
    정확도 (Accuracy)전체 중에서 맞힌 비율(TP+TN) / 전체
    
      
    정밀도 (Precision)모델이 "암"이라고 한 것 중 진짜 암의 비율 — 헛다리 짚지 않기TP / (TP+FP)
    
      
    재현율 (Recall)실제 암 환자 중 모델이 잡아낸 비율 — 놓치지 않기TP / (TP+FN)
    
    
    
    
    
      🚨 정확도의 함정 — 1,000명 중 실제 암 환자가 10명뿐이라면? 모두에게 "정상"이라고 찍는 엉터리 모델도 정확도 99%가 나옵니다. 하지만 재현율은 0% — 살려야 할 환자를 전부 놓친 거죠. (⑪ 심화 도전 Q4가 바로 이 이야기!)
    
    
  
    
  
    
    👨‍🏫 선생님의 한 줄 요약
    
    놓치면 큰일 나는 문제(암 진단·화재 경보)는 재현율, 헛다리가 큰일인 문제(스팸 필터 — 중요 메일이 스팸함행!)는 정밀도를 챙겨라!
  
    
  
    

  
    📝 검증 데이터(Validation Data) — 하이퍼파라미터의 '모의고사'
    
  
    
    
    
      공부에 비유하면 데이터셋은 세 가지로 나뉩니다. 교과서·문제집으로 공부(훈련)하고,
      모의고사로 공부 방법을 점검(검증)한 뒤, 수능으로 진짜 실력을 평가(테스트)하죠.
      하이퍼파라미터(학습률, 레이어 수, epochs…)는 말하자면 '공부 방법'이고,
      검증 데이터는 그 공부 방법이 올바른지 중간 점검하는 모의고사입니다.
    
    

    
    
    
      
    훈련 Train
    교과서·문제집 (예: 60%)
    
      
    검증 Val
    모의고사 (20%)
    
      
    테스트 Test
    수능 (20%)
    
    
    
    
    ※ 비율은 예시예요. 케라스에서는 validation_split=0.2처럼 훈련 데이터의 일부를 떼어 검증에 씁니다.
    

    
    
      
    구분실생활 비유역할
    
      
    훈련 데이터
    (Train Data)    		교과서와 문제집모델의 가중치(Weight)와 편향(Bias)을 실제로 학습시키는 데이터
    
      
    검증 데이터
    (Validation Data)    		학원 모의고사학습 도중 중간 점검을 통해 최적의 하이퍼파라미터(학습률, 레이어 수 등)를 결정하는 데이터
    
      
    테스트 데이터
    (Test Data)    		최종 수능 시험모든 학습과 튜닝이 끝난 후, 모델의 진짜 실력(일반화 성능)을 최종 평가하는 데이터
    
    
    

    
    
      🚨 왜 훈련 데이터로 하이퍼파라미터를 정하면 안 될까? — 과대적합(Overfitting)
    
      훈련 데이터 점수만 보고 공부 방법을 고르면, 모델은 문제집을 통째로 외워버리는 길을 택합니다.
      문제집(훈련)은 100점인데 처음 보는 시험(테스트)은 엉망이 되는 것 — 이것이 과대적합이에요('과적합'이라고 부르는 책도 있어요).
      검증 데이터는 모델이 단순 암기가 아니라 원리를 이해(일반화)하고 있는지 감시하는 기준이 됩니다.
      모의고사 점수가 떨어지기 시작하면 "아, 지금 외우고 있구나"를 알아채는 거죠.
    
    

    
    💡 아래 컴파일/학습 코드의 validation_split=0.2가 이 '모의고사'를 만드는 부분이에요. 위 플립카드 사전에서도 확인해 보세요!

    Q1. 훈련 정확도는 99%인데 검증 정확도가 65%로 뚝 떨어졌다. 무슨 일이 일어난 걸까?
    
    
    👉 과대적합. 모델이 훈련 데이터를 통째로 암기해 일반화에 실패한 상태. 에폭을 줄이거나 드롭아웃·규제를 추가한다. (양궁 시뮬의 Epoch 3 상태!)
    
  
    
  
    
    
    Q2. "여러 모델의 예측값을 다시 새 데이터로 삼아 2단계 모델이 학습"하는 앙상블 기법은?
    
    
    👉 스태킹(Stacking). 배깅(동시)·부스팅(이어받기)과 달리 예측 패턴 자체를 한 번 더 학습한다.
    
  
    
  
    
    
    Q3. 다중 클래스 분류의 출력층 활성화 함수와, 그 확률들의 합은?
    
    
    👉 소프트맥스(Softmax), 확률의 합은 항상 1(100%). 이진 분류라면 시그모이드를 쓴다.
    
  
    
  
    
    
    Q4. 어떤 암 진단 AI가 정확도(accuracy) 99%를 기록했다. 병원장은 기뻐했지만, 개발자는 오히려 얼굴이 굳었다. 왜일까?
    
    
    👉 정확도의 함정(불균형 데이터). 전체 환자 중 실제 암 환자가 1%뿐이라면, 모든 사람을 무조건 '정상'이라고 찍는 엉터리 모델도 정확도 99%가 나온다. 정작 살려야 할 암 환자를 전부 놓치는(재현율=0) 최악의 모델일 수 있는 것. 그래서 정확도 하나만 보면 안 되고, 정밀도(Precision)·재현율(Recall)을 함께 봐야 한다. — "무엇을 맞혔나"보다 "무엇을 놓쳤나"가 더 중요할 때가 있다.
    
  
    
  
    
    
    Q5. 훈련 정확도 100%, 테스트 정확도도 100%였던 모델을 실제 서비스에 배포했더니 성능이 형편없었다. 과대적합도 아닌데, 무엇을 놓친 걸까?
    
    
    👉 데이터 편향 & 분포 이동(Distribution Shift). 훈련·테스트를 같은 데이터에서 나눴다면, 그 데이터 자체가 현실을 대표하지 못할 때 둘 다 높은 점수가 나와도 소용없다. 예컨대 낮 시간대 사진만으로 학습한 자율주행 AI는 밤·비 오는 날 무너진다. 테스트 점수는 "수집한 데이터와 비슷한 상황"에서의 실력일 뿐 — 좋은 모델보다 현실을 닮은 좋은 데이터가 먼저다.
